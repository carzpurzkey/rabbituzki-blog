¶3.2.1 scanf的工作方法（P30）

  对于格式串中的第一个转换说明，scanf函数从输入的数据中定位适当类型的项，并在必要时跳过空格。然后，scanf函数诗篇数据项，并且在遇到 不可能属于此项的字符（笔记：如果不可能属于此项的字符为本次读取的第一个字符，则scanf立即停止。否则将此字符放回原位，在下次读取时作为第一个读取的字符。参考P31例子） 时停止。如果读入数据项成功，那么scanf函数会继续处理格式串的剩余部分；如果某一项不能成功读入，那么scanf函数将不再查看格式串的剩余部分（或余下的输入数据）而立即返回。

  在寻找数的起始位置时，scanf函数会忽略空白字符。因此，我们可以把数字放在同一行或者分散在几行内输入。

¶3.2.3易混淆的printf函数和scanf函数（P32）

  printf格式串结党以\n结尾，但是在scanf格式串末尾旋转换行符通常是一个坏主意。对scanf函数来说，格式串中的换行符等价于空格，两者都会引发scanf函数提前进入下一个非空白字符。例如，如果格式串是"%d\n"，那么函数将跳过空白字符，读取一个整数，然后跳到下一个非空白字符处。像这样的格式串可能会导致交互式程序一直“挂起”直到用户输入一个非空白字符为止。

¶第三章FAQ（P33）

  %i与%d区别：%i可以为十进制、八进制（0开头）、十六进制（0x或0X开头）的数，%d只识别十进制数。

¶第五章 选择语句

¶5.2 if语句

¶5.2.4 “悬空else”的问题（P56）

  C语言遵循的规则是else子句应该属于离它最近的且还未和其它else匹配的if语句。

¶5.2.7 C99中的布尔值（P58）

  在C99中提供了_Bool型，_Bool是无符号整型，但只能赋值为0或1，一般来说，往_Bool变量中存储非零值会导致变量赋值为1。
  除了_Bool类型的宝座，C99还提供了一个新的头<stdbool.h>。该问提供了bool宏来代表_Bool，true来代表1，false来代表0。

¶5.3 switch语句

  1. switch后边必须跟着由圆括号括起来的整型表达式，不能用浮点数和字符串。
  2. case后跟常量表达式，如5 + 10。

¶第五章FAQ（P63~P64）

  1. 当int型和float型的值混合在一个条件表达式中时（如i > 0 ? i : f）表达式类型为float型。如果i > 0为真，那么变量i转化为float型后的值就是表达式的值。
  2. C89标准指出，以下划线后跟一个大写字母开头的名字是保留字，程序员不应该使用。

¶第七章 基本类型

¶7.1整数类型（P89)

  short int、int、long int取值范围有可能是一样的。但C标准规定int取值范围不能比short int小，long int取值范围不能比int小。

¶7.1.4整数溢出（P91）

  1. 有符号整数溢出行为未定义。
  2. 无符号整数溢出为对2n取模，其中n是用于存储结果的位数。例如，如果对无符号的16位数65535加1，其结果可以保证为0。

¶7.4类型转换（P100）

  当发生下列情况时会进行隐匿转换。

  • 当算术表达式或逻辑表达式中操作数的类型不相同时。（C语言执行所谓的常用算术转换）
  • 当赋值运算符右侧表达式的类型和左侧变量的类型不匹配时。
  • 当函数调用中的实参类型与其对应的形参类型不匹配时。
  • 当return语句中表达式的类型和函数返回值的类型不匹配时。

¶7.4.1常用算术转换（P101）

  1. 任一操作数是浮点类型的情况。

            double

             float

  2. 两个操作数的类型都不是浮点类型的情况。

            unsigned long int
                 ^

              long int
                 ^
                 |
           unsigned int
                 ^
                 |
                int

  有一种特殊情况，只有在long int类型和unsigned int类型长度（比如32位）相同时才会发生。在这类情况下，如果一个操作数的类型是long int，而另一个的类型是unsiged int，那么两个操作数都会转换成unsigned long int类型。C语言把（类型名）视为一元运算符。一元运算符的优先级高于二元运算符，所以编译器会把表达。

  当把有符号操作数和无符号操作数组合时，把有符号操作数“转换”成无符号的值。转换过程中需要加上或者减去n + 1的倍数，其中n是无符号类型能表示的最大值。这条规则可能会导致某此隐蔽的编程错误。

  假设int类型的变量i的值为-10，而unsigned int类型的变量u的值为10。如果用<运算符比较变量i和变量u，那么期望的结果应该是1（真）。但是，在比较前，变量i转换成为unsigned int类型。因为负数不能被表示成无符号整数，所以转换后的值不再为-10，而是加上4294967296的结果（假定4294967296是最大的无符号整数），即4294967286。因而i < u比较的结果将为0。有些编译器会在程序试图比较有符号与无符号时给出一条类似“comparison between signed and unsigned”的警告消息。

  由于此类陷阱的存在，所以最好尽量避免使用无符号整数，特别是不要把它和有符号整数混合使用。
